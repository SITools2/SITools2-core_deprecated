Registry handles all applications that SITools2 hosts. From this application, the administrator can see the details, delete, enable or disable an application as well as change authorizations.

Registry

Show application's details

  1. Select the application you want to see.
  2. Click on Details to see details.
  3. Click on Close to close the window.

Authorizations

  1. Select the application for which you want to see authorizations.
  2. Click on Authorizations.
  3. A window opens an authorization panel with the following functions:
    • Create a role: click on Create to add one or more roles for this application. Select all roles you want to add then click on OK to validate.
      One or more lines appear in the table then you can select all authorizations you want to set for the role:
      • all : can do everything
      • get : can consult resources without modify them
      • post : can create resources
      • put : can modify resources
      • delete : can delete resources
      • head :
      • options :
    • Delete a role : select a role in the list, click on Delete then click on OK.

Enable an application

  1. Select the application on the list
  2. Click on Enable to enable the application, the applicaiton available

Disable an application

  1. Select the application on the list
  2. Click on Disable to disable the application, the application is no longer available

Delete an application

  1. Select the application on the list
  2. Click on Delete to delete the application, the application is no longer available

Application API

For an easier authorization of an application according to both role and HTTP methods (get post, put, ...), the administrator can get the API description of each application by clicking on the question mark

Note:

If no role is set for one application, then a default authorization is applied (see Authorizations in Security Management).
