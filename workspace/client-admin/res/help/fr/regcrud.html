When a user asks for an account in the archive, he must fill a set of information. This information is then transferred to the application in the “Registers” panel. The administrator must then validate it to create a user account. A user account provides some special capabilities to the user:

  • Accessing to asynchronous services
  • Having its own user storage
  • Saving the user preferences
  • Accessing to restricted projects, services or datasets

The “Registers” panel shows the list of user accounts waiting for a validation. The panel displays the login, first name, last name and email that the user filled during the registration step. The administrator can delete a user account, edit his information or validate it. When a user is validated, the user information is stored in the user table and the system sends to the newly created user a confirmation email.

Validate a user

  1. Click on a row to select a user information
  2. Click on Validate.

A new user account is then created in the archive. A confirmation email is automatically sent to the user.

Edit a user account

  1. Find the user account by entering the last name in the search field
  2. Double click on it or click on Edit
  3. Edit the user information.

The administrator can change all fields related to the user. He can also add some properties based on keyword/value.

Delete a user account

  1. Find the user account by entering the last name in the search field
  2. click on it and click on Delete.
  3. The user information will be definitively removed from the archive as well as his user space
