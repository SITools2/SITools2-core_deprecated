Solr Type

This is the type used by SolR for the indexing.
text  :
  • This is the default type
  • it cuts the term  on every case change
  • it cuts the term  on changes from numeric to alphabet.
  • It concatenates numbers ( 125-25 = 12525)
  • it concatenates words (wi-fi = wifi).
  • It keeps the origininal term.
  • It apply a list of unindexed words and synonims
 text_ws :
  • It splits only on whitespace for exact matching of terms
string
  • You have to use this type for unique key otherwise it won't work.

    Note:
  1. The string type is not analyzed, but indexed/stored verbatim, which means that it can"t be queried but it will be returned
  2. You can also used any other types defined in schema.xml such as date, boolean, int ...
