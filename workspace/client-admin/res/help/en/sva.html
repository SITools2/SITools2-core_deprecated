SVA are services that can be attached in a dataset. They can be added to Sitools2 as a plugin module extensions (fr.cnes.sitools.extensions.jar).

Developers of a SVA may combine the use of filters and converters that are already defined on the dataset to fit their own query and return results with their own management synchronous or asynchronous.

Associate a SVA

  1. Select the dataset in the ComboBox.
  2. Click on Create.

    SVA class panel

    1. Choose a SVA that you want to apply

    SVA details panel

    1. Label : the label
    2. Run type :

    -          SVA_FORCE_RUN_SYNC : the SVA will execute as a synchronous request

    -          SVA_FORCE_RUN_ASYNC : the SVA will exectute as an asynchronous request

    -          SVA_DEFAULT_RUN_SYNC : The SVA will ask the user the type of REQUEST, and by default an application will run synchronously

    -          SVA_DEFAULT_RUN_ASYNC : The SVA will ask the user the type of REQUEST, and by default an application will run asynchronously

    1. Method : Querying the SVA will execute a POST or a GET request.
    2. Specific Folder : check specificFolder to create a new folder to store files for each SVA tasks

    Field Parameters mapping panel

    1. Associate required column to parameters
    2. Fill in required internal parameters

     

  3. Click on Ok

Edit a SVA

  1. Select a SVA in the list
  2. Click on Edit and edit the SVA details
  3. Click on Ok

    Delete a SVA

    1. Select a SVA in the list
    2. Click on delete
    3. Click on Yes to confirm your choice.

    Activate a SVA

    1. Select a SVA in the list
    2. Click on Enable

    Desactivate a SVA

    1. Select a SVA in the list
    2. Click on Disable
